## ✅ Naprawione w tej sesji

### CSS — krytyczny bug ukrywania nieaktywnych templates

**Problem:** Reguły w `src/styles/global.css` ukrywały tylko parę
`klasyk ↔ marka`. Selektory dla `magazyn` i `stadion` nie istniały. Skutek:
gdy aktywny był `klasyk`, użytkownik scrollował i widział **klasyka + magazyn
+ stadion** sklejone pionowo na jednej stronie. To wyglądało jak całkowicie
zepsuty layout (Tomek początkowo myślał że „ruszałem klasyka" — w rzeczywistości
widział klasyka z naklejonymi pod nim resztami).

**Fix:** rozszerzono blok do pełnej macierzy 4 × 4 (12 selektorów `display: none !important`
dla wszystkich kombinacji). Plik: `src/styles/global.css`, linie 167–181.

Weryfikacja: po reload `document.querySelectorAll('[data-template-only]')` zwraca
4 elementy, **z czego dokładnie 1 ma `offsetHeight > 0`** dla każdego ustawienia
`localStorage.wks-template`.

## 🔴 Do zrobienia — KRYTYCZNE (przed pokazaniem zarządowi)

### 1. Header (i footer) NIE reaguje na template Magazyn (dark mode)

**Najboleśniejszy problem wizualny.** Magazyn ma czarne body + jasne karty,
ale `SiteHeader` nadal jest jasny zielony (z białą belką nawigacji). Przy
wejściu na np. `/aktualnosci` w trybie magazyn cała strona jest dark, a nad
nią siedzi standardowy jasny pasek. Wygląda jak dwa różne projekty sklejone
razem. Footer ma analogiczny problem.

**Plan fix:** dorzucić do `src/styles/global.css` reguły:

```css
html[data-template="magazyn"] header.site-header,
html[data-template="magazyn"] header.site-header * {
  background-color: #000 !important;
  border-color: rgb(30 41 59) !important;
}
html[data-template="magazyn"] header.site-header a { color: rgb(226 232 240); }
html[data-template="magazyn"] header.site-header a:hover { color: rgb(var(--club-secondary)); }
```

Analogicznie dla `footer`. Należy najpierw obejrzeć selektor headera
(`src/components/SiteHeader.astro` — sprawdzić czy ma klasę `site-header`,
jeśli nie — dodać).

**Estymacja:** 30–45 min (z testowaniem na wszystkich podstronach).

## Aktualne reguły CSS dla template-switching (referencja)

Plik: `src/styles/global.css`

- **Linie 100–162** — keyframes (`marka-marquee`, `marka-pulse`, `marka-fadeup`,
  `marka-orb-a/b`, `stadion-lightbar`) + utility classes + `prefers-reduced-motion`.
- **Linie 167–181** — macierz 4 × 4 ukrywająca nieaktywne `[data-template-only]`
  (świeżo naprawiona).
- **Dalej** — bloki per template (`html[data-template="marka"]`, `magazyn`,
  `stadion`) ze specyficznymi nadpisami: gradienty headera podstron,
  border-radius buttonów, dark mode dla magazyn (body/main/.bg-white/.card/
  .section-title/.section-kicker/.prose-club/footer + drop-cap), animowane
  paski stadion.

System włączania: `src/templates/init.astro` (inline script w `<head>`,
zapobiega FOUC) + `src/templates/TemplateSwitcher.astro` (floating widget).
Lista templates: `src/templates/registry.ts`.

**Łatwa odinstalacja całego systemu templates:** usunąć folder `src/templates/`,
usunąć importy `<TemplateInit/>` i `<TemplateSwitcher/>` z `BaseLayout.astro`,
usunąć blok CSS „System szablonów" z `global.css`, w `src/pages/index.astro`
zostawić tylko zawartość bloku `<div data-template-only="klasyk">` (bez
wrappera) — tak żeby wrócić do oryginalnego klasyka.
